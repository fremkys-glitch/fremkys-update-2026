# Airtable Field Mapping - FREMKYS Orders System

## Overview
This document details the complete field mapping between the FREMKYS order system and Airtable Orders table.

---

## Complete Field List

### Fields Sent to Airtable

| Field Name | Type | Required | Description | Example |
|------------|------|----------|-------------|---------|
| `Order Number` | String | Yes | Unique order identifier | FREMKYS-1733427600000 |
| `customer_first_name` | String | Yes | Customer's first name | Ahmed |
| `customer_last_name` | String | Yes | Customer's last name | Benzaid |
| `customer_email` | String | No | Customer's email address | ahmed@example.com |
| `customer_phone` | String | Yes | Customer's phone number | +213 555 123 456 |
| `Shipping Address` | String | Yes | Full shipping address | 123 Rue de la Paix |
| `shipping_city` | String | Yes | City name | Algiers |
| `shipping_wilaya` | String | Yes | Province/State | Alger |
| `delivery_type` | String | Yes | Delivery method | توصيل منزلي / توصيل للمكتب |
| `notes` | String | No | Customer notes | Handle with care |
| `Items` | Long Text (JSON) | Yes | Complete items array as JSON | [{"id":"1","name":"Product","price":5000,"quantity":1,"size":"M"}] |
| `product_name` | String | Yes | Formatted product names | Product 1 (x2), Product 2 (x1) |
| `product_size` | String | Yes | Formatted product sizes | M (x2), L (x1) |
| `subtotal` | Number | Yes | Order subtotal amount | 10000 |
| `shipping_fee` | Number | Yes | Shipping cost | 600 |
| `Total` | Number | Yes | Total order amount (subtotal + shipping) | 10600 |
| `status` | String | Yes | Order status | طلب جديد |

### Fields Calculated by Airtable (DO NOT SEND)

These fields are computed automatically by Airtable formulas or timestamps:

| Field Name | Type | Auto-Generated By | Description |
|------------|------|-------------------|-------------|
| `ID` | Auto Number | Airtable | Unique record ID (auto-incremented) |
| `Full Customer Name` | Formula | Airtable | Concatenation of first + last name |
| `Confirmation Date` | Date/Time | Airtable | Timestamp when order confirmed |
| `Created At` | Created Time | Airtable | Record creation timestamp |
| `Updated At` | Last Modified Time | Airtable | Record last update timestamp |

## Airtable Column Setup Requirements

### 1. Text Fields
Set these columns to **Single line text** or **Long text**:
- Order Number (Single line text)
- customer_first_name (Single line text)
- customer_last_name (Single line text)
- customer_email (Single line text or Email)
- customer_phone (Single line text or Phone)
- Shipping Address (Long text)
- shipping_city (Single line text)
- shipping_wilaya (Single line text)
- delivery_type (Single line text or Single select)
- notes (Long text)
- Items (Long text)
- product_name (Long text)
- product_size (Single line text)
- status (Single line text or Single select)

### 2. Number Fields
Set these columns to **Number** type:
- subtotal (Number, precision: 2)
- shipping_fee (Number, precision: 2)
- Total (Number, precision: 2)

### 3. Auto-Generated Fields
- ID (Auto number)
- Created At (Created time)
- Updated At (Last modified time)

### 4. Formula Fields
- Full Customer Name: `{customer_first_name} & " " & {customer_last_name}`
- Confirmation Date: (Set your formula as needed)

## Common Issues and Solutions

### Issue: UNKNOWN_FIELD_NAME
**Cause:** Field name mismatch between code and Airtable
**Solution:**
1. Check Airtable column name is EXACT match
2. Remember: `Order Number` (with space) vs `order_number` (with underscore)
3. Some fields use Title Case, others use snake_case

### Issue: INVALID_VALUE_FOR_COLUMN
**Cause:** Wrong data type
**Solution:**
1. Ensure subtotal, shipping_fee, Total are Number type
2. Ensure Items is Long text type (not Single line text)

### Issue: Items field too short
**Cause:** Single line text has character limit
**Solution:** Change Items column to Long text type
